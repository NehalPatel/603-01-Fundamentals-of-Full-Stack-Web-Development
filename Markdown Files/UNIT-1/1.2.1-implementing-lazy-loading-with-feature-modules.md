## Learning Objectives

- Lazy load standalone components using `loadComponent`
- Understand legacy module-based lazy loading via `loadChildren`
- Optimize bundle sizes and configure preloading

## Overview

- Angular 17 recommends standalone lazy loading for new apps
- Legacy `NgModule` lazy loading remains for existing codebases

## Standalone Lazy Loading

```ts
import { Routes } from '@angular/router';

export const routes: Routes = [
  { path: 'admin', loadComponent: () => import('./features/admin/admin.component').then(m => m.AdminComponent) }
];
```

## Legacy Module Lazy Loading

```ts
import { Routes } from '@angular/router';

export const routes: Routes = [
  { path: 'legacy', loadChildren: () => import('./legacy/legacy.module').then(m => m.LegacyModule) }
];
```

## Preloading Strategy

```ts
import { provideRouter, withPreloading } from '@angular/router';
import { PreloadAllModules } from '@angular/router';

provideRouter(routes, withPreloading(PreloadAllModules));
```

## Troubleshooting & Pitfalls

- Ensure components are declared `standalone: true` when using `loadComponent`
- Avoid large shared imports in lazy features to prevent duplication
- Verify that guards and resolvers are registered at the route level
