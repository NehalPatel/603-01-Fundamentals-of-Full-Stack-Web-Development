# Nodemon and Project Structuring

## Learning Objectives

- Use nodemon for auto-restart in development
- Organize code into routes/controllers/services
- Configure `npm` scripts for dev and prod

## Overview

- Nodemon watches files and restarts the server on changes
- Feature-based folder structure improves maintainability

## Commands

```bash
npm i -D nodemon
```

```json
{
  "scripts": {
    "dev": "nodemon src/index.js",
    "start": "node src/index.js"
  }
}
```

## Folder Structure

```text
src/
  routes/
  controllers/
  services/
  index.js
```

## Practical Exercises

### Exercise

- Move inline route handlers into controllers and import them from routes

### Solution

```js
export const list = (req, res) => res.json([])
```

```js
import { Router } from 'express'
import { list } from '../controllers/items.js'
const r = Router()
r.get('/', list)
export default r
```

## Troubleshooting & Pitfalls

- Use `--ext js,ts,json` for nodemon when mixing file types
- Avoid circular imports between controllers and services
